# Information Security Risk Assessment / Analysis
- Involves identifying and assessing risks related to CIA of Information systems and resources.
## Why Assess Information Security Risk?
- In order to protect something, one must first **identify**;
1. What to protect(Assets)
2. What to protect from
## An Information Security Function
- The **organizational structure, roles, and processes** established within an entity to **manage, maintain, and continually improve** the organization's information security posture and controls.
- An Information Security Function should be able to use Information Security Risk Assessment to achieve 3 primary objectives.
1. Determine Safeguards to secure IS that store and process information.
2. Allow an organization to comply with internal and external requirements.
3. Make well informed decisions.

### 4. Identify ImpactğŸ”¥
- This step is a part of an **impact-likelihood matrix** ; which will return risk ratings.
- This can be done in two main ways;
	1. Quantitative risk assessments
	2. Qualitative risk assessments
#### Quantitative  Risk Assessment
- Uses numerical values to estimate potential losses / impact.
- Provides an **object-data** driven approach.
- **Often used in financial and high risk industries.** 
	1. Three Point Estimate
	2. Decision Tree Analysis
	3. Expected Monetary Value 
	4. Monte Carlo Analysis
#### Qualitative Risk Assessment
- Uses subjective ratings to determine impact.
- Relies on - 
	**Expert Judgement , historical data and Predefined risk matrices**.
- **Suitable for** **organizations without sufficient numerical data**.
	1. Historical Data 
	2. SWOT analysis
	3. Brainstorming
	4. Risk Matrix
	5. Delphi method
	6. Scenario Analysis
- Objective - Produce a measurement for impact
- Outcome - Measurement metrics
